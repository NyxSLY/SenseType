# 折腾了一个 Windows 本地语音输入工具，按快捷键说话直接打字，不联网

起因是打字太慢，想用语音输入。

试了一圈现有方案都不太满意：
- **Win+H**（Windows 自带的）：必须联网，中文识别效果一般
- **讯飞语音输入**：要登录、要联网、免费版有限制
- GitHub 上搜了搜，开源的基本都是基于 Whisper，中文不是它的强项，速度也慢

后来发现阿里开源了一个语音识别模型 **SenseVoice-Small**，中文效果比 Whisper 好不少，速度也快很多（10 秒的音频推理只要 70ms，Whisper 要 1 秒多）。

但 SenseVoice 只是个模型，不能直接当工具用。于是我花了几天时间把录音、快捷键、识别、自动粘贴这些串起来，做成了一个能在 Windows 上日常用的语音输入工具：

**按快捷键 → 说话 → 松开 → 文字自动出现在光标位置**

浏览器、Word、VS Code、微信、记事本，光标在哪就输到哪。

---

## 大概能干嘛

- 完全离线，不传任何数据
- 中文效果不错，中英混合也能认
- 自动加标点，不用自己说"逗号""句号"
- 有 GPU 自动用 GPU，没有就走 CPU，不用手动配
- 屏幕底部有个浮动状态条，录音时能看到音量
- 双击一个文件就能启动，不用敲命令行

## 不足

就是个自用的小东西，比较简陋：

- 只支持 Windows
- 需要装 Python 环境（写了比较详细的安装指南）
- 全局快捷键需要管理员权限
- 模型首次运行要下载大概 400MB

识别能力是阿里 SenseVoice 的，我就是在外面套了一层，把录音、快捷键监听、粘贴到光标这些事给处理了，让它能当个正常工具用。
